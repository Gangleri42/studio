# SeedHammer Studio

Two static web apps for composing plates and sending them to a SeedHammer II.

- **Studio** (the editor) — site root: <https://gangleri42.github.io/studio/>
  Compose plates from text, rich text, and SVG graphics, preview the exact
  engraving, and send it over NFC. Installable as a PWA.
- **Text composer** — <https://gangleri42.github.io/studio/textplate/>
  A minimal plain-text plate composer.

## Architecture

This repo is a **static consumer**. It ships HTML, JS, icons, and one
generated data file (`glyphs.js`); the deploy copies files and runs no build
toolchain. The firmware repo ([`Gangleri42/seedhammer`](https://github.com/Gangleri42/seedhammer))
is the source of truth for anything derived from the firmware.

## glyphs.js

`glyphs.js` holds the font/glyph outlines the apps render. It is generated by
the firmware's `cmd/textplate` from the firmware font tables, then committed
here (once at the root for Studio, once under `textplate/` for the composer;
the two are byte-identical).

`.github/workflows/glyphs-drift.yml` guards it: on every change it regenerates
`glyphs.js` from a pinned firmware commit and byte-compares. This catches a
committed copy going stale against the firmware. To update the font:

1. In the firmware repo: `go run ./cmd/textplate glyphs.js`.
2. Copy the result over both `glyphs.js` and `textplate/glyphs.js` here.
3. Set `FIRMWARE_REF` in `glyphs-drift.yml` to the firmware commit you
   generated from.
4. Commit the three changes together.

The pin is a commit SHA, not `main`, so unrelated firmware edits never turn
the check red.

## Sending to the device

The editor writes over NFC two ways:

- **Android Chrome:** direct Web NFC, no helper needed.
- **Desktop:** a local bridge (`cmd/nfc-bridge` in the firmware repo) exposes
  a USB NFC reader at `http://127.0.0.1:8787`. The editor talks to it there.
  The bridge allow-lists this site's origin (`https://gangleri42.github.io`)
  for the cross-origin call; run it locally to enable desktop sending.

## Deploy

GitHub Pages, via `.github/workflows/pages.yml` on push to `main`. The editor
publishes at the site root, the composer under `textplate/`.
